PDF Output (CV + Cover Letter)
------------------------------

Downloadable A4 PDFs for the CV and cover letter are **implemented** (branch
``editorial-resume-theme``). Pre-generated static files live in
``homepage/static/resume/``:

* ``katharina-lebenslauf.pdf`` — one-page A4 CV
* ``katharina-anschreiben.pdf`` — one-page A4 cover letter

Generation uses ``manage.py render_resume_pdfs`` (headless Chromium via
Playwright): runs against a live server, checks HTTP status and page structure
before writing, and fails loudly on bad tokens or missing pages. The
``just render-pdfs`` recipe starts its own isolated server on port 8765.

Download links in the CV "Resources" rail point to the committed static files;
they are screen-only (hidden in print via the ``@media print`` block in
``print.css``). Re-run the command after content edits and commit the updated
PDFs.

**Remaining:** PDFs can go stale after editorial content edits — no automatic
re-generation hook exists yet. Consider wiring ``render-pdfs`` into the deploy
pipeline or adding a CI check.
